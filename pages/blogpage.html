Why Documentation is Crucial

What is Documentation

Documentation is an explanation for the code you create. In order to make your code more understandable for someone reading it (i.e., open-source contributors, project manager, co-worker) documentation helps the reader understand WHY your code was implemented. Documentation can also be done in the form of using short and descriptive names for your elements.

Benefits of naming elements and variables

By providing a name that describes your elements and functions, this elimnates the need to use comments. Reducing the need for comments also helps with readability within your documentations since its less clutter within your code. The objective to documentation is to make it so understandable that someone whose not a programmer can also follow along with what you created. Your documentation can act as a rubric for your reader.

What can you add to your technical documentation

You can separate your documentation from the source code and provide a detailed essay of some sort about the features implemented into your project.

Information to include:
  1. Objective of your project
  2. Who will it benefit the most (target audience)
  3. Programming languages used
  4. Future of the project
